ℹ️ Thông Tin Chi Tiết

📋 Phương Án 1: Tách Bảng Predictions

Từ: 1 bảng 52 cột → Thành: 6 bảng chuyên biệt

  • predictions - Core data (7 cột)
  • prediction_patient_info - Thông tin bệnh nhân (6 cột)
  • prediction_results - Kết quả phân loại (10 cột)
  • prediction_analysis - Phân tích hình ảnh (11 cột)
  • prediction_doctor_review - Xét duyệt bác sĩ (4 cột)
  • prediction_processing_log - Nhật ký xử lý (4 cột)

👥 Phương Án 2: Tách Bảng Users

Từ: 1 bảng 14 cột → Thành: 3 bảng chuyên biệt

  • users - Xác thực (5 cột)
  • user_profiles - Thông tin cá nhân (5 cột)
  • user_security_logs - Bảo mật (5 cột)

✨ Lợi Ích

  • ✅ Quản lý dữ liệu tốt hơn - Dễ bảo trì, cập nhật từng phần
  • ✅ Hiệu suất cao hơn - Query chỉ lấy dữ liệu cần thiết
  • ✅ Tái sử dụng dữ liệu - Một bệnh nhân/bác sĩ có thể có nhiều dự đoán
  • ✅ Phân quyền chi tiết - Kiểm soát truy cập từng bảng
  • ✅ Linh hoạt mở rộng - Dễ thêm bảng mới mà không ảnh hưởng hiện tại

📝 Ký Hiệu

  • PK - Primary Key (Khóa chính)
  • FK - Foreign Key (Khóa ngoại)
  • UK - Unique Key (Khóa duy nhất)
  • 1:∞ - Quan hệ Một-Nhiều
  • 1:1 - Quan hệ Một-Một
